# 🚀 คู่มือการทดสอบ API (API Testing Guide)

ไฟล์นี้จะสอนวิธีการทดสอบ API ของระบบ Skill Badge Platform ของเราแบบละเอียดสุดๆ ครับ โดยจะเน้นให้เข้าใจว่า "API เส้นนี้ทำอะไร?", "ต้องใช้สิทธิ์ (Login) ไหม?", "ใส่ข้อมูล (JSON) แบบไหน?" และ "จะทดสอบมันยังไง?" ครับผม

---

## 🛠 วิธีทดสอบ API (เริ่มจากตรงนี้)

วิธีที่ง่ายและดีที่สุดโปรเจกต์นี้คือการใช้ **Swagger UI** เพราะเราสร้างไว้ให้แล้ว!
1. รันเซิร์ฟเวอร์ Backend (`npm run dev` หรือ `bun run dev`)
2. เปิดเบราว์เซอร์ไปที่: `http://localhost:3000/swagger`
3. คุณจะเจอหน้าเว็บที่มีรายชื่อ API ทุกเส้นรวมอยู่ สามารถกดปุ่ม **Try it out** แล้วกรอกข้อมูลทดสอบได้เลย!

### 🔑 คำแนะนำเรื่องการ Login (Bearer Token)
API หลายเส้นบังคับว่า **"ต้อง Login"** ซึ่งแปลว่าเราต้องแนบ `Token` เข้าไปใน Header ตอนยิง API ด้วย
ถ้าคุณทดสอบผ่านหน้าเว็บ (Frontend) ระบบจะแนบไปให้อัตโนมัติ แต่ถ้าทดสอบผ่าน Swagger / Postman ต้องทำแบบนี้:

1. ตอนเชื่อมต่อฝั่ง Frontend กับ Firebase (หรือการ Login) เราจะได้ **Firebase ID Token** มา (สามารถกดดูได้ที่ Network/Console ตอน Login ผ่านเว็บ)
2. เอา Token นั้นมาใส่ใน Swagger โดยกดปุ่ม **Authorize** 🔓 ขวาบน 
3. วาง Token ลงไป (ถ้าไม่ได้ให้พิมพ์คำว่า `Bearer ` นำหน้าเว้นวรรค 1 ที ตามด้วย Token) แล้วกด Save
4. ทีนี้เวลายิง API เส้นที่ "ต้อง Login" มันก็จะผ่านแล้วครับ!

## 📋 รายการ API ทั้งหมด (API Endpoints) พร้อมตัวอย่าง JSON รูปแบบคำขอ

นี่คือโพยว่าแต่ละเส้นมีไว้ทำอะไรบ้างครับ แบ่งตามหมวดหมู่เลย

### 1. 🛡️ หมวด Authentication (`/auth`)
จัดการเกี่ยวกับการยืนยันตัวตนและการ Login

*   `POST /auth/github/callback`
    *   **ทดสอบอะไร:** นำ Code จาก GitHub OAuth มาแลกเป็นหน้าต่างเข้าสู่ระบบ (Custom Token)
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "code": "ตัวอักษรของคีย์ที่ได้จาก URL หลังจาก Github Redirect กลับมา"
      }
      ```
*   `POST /auth/verify`
    *   **ทดสอบอะไร:** ตรวจสอบว่า Token ที่เรามีอยู่มันยังไม่หมดอายุใช่มั้ย
    *   **ต้อง Login ไหม:** ✅ ต้องใช้ Bearer Token
*   `DELETE /auth/logout`
    *   **ทดสอบอะไร:** ลบ Session/Token ออกจากระบบ (Logout)
    *   **ต้อง Login ไหม:** ✅ ต้อง Login

### 2. 👤 หมวดผู้ใช้งาน (`/users`)
จัดการข้อมูล Profile พื้นฐาน ค้นหาคน และดูข้อมูลพอร์ตฟอลิโอ

*   `GET /users/search`
    *   **ทดสอบอะไร:** ค้นหาผู้ใช้งานในระบบ (เช่น พิมพ์ชื่อ หรือ สถานที่)
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `GET /users/recommendations`
    *   **ทดสอบอะไร:** ดึงรายชื่อ Developer โปรไฟล์น่าสนใจที่ระบบแนะนำให้รู้จัก
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `GET /users/me` (และ `/users/profile`)
    *   **ทดสอบอะไร:** ดึงข้อมูลส่วนตัวของเราเอง
    *   **ต้อง Login ไหม:** ✅ ต้อง Login 
*   `POST /users/sync`
    *   **ทดสอบอะไร:** นำข้อมูลจาก Firebase Auth มาเซฟลงฐานข้อมูลหลักของเรา (ใช้ตอนสมัครหรือเข้าระบบครั้งแรก)
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "uid": "รหัส UID ผู้ใช้งานที่ได้มาจาก Firebase",
        "email": "user@example.com",
        "displayName": "John Doe",
        "photoURL": "https://url-รูปภาพ..."
      }
      ```
*   `PUT /users/me`
    *   **ทดสอบอะไร:** ทดสอบการอัพเดตข้อมูลโปรไฟล์ (เช่น เปลี่ยนชื่อ, เปลี่ยน bio, เพิ่มลิงก์ linkedin)
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):** *(ส่งเฉพาะฟิลด์ที่ต้องการเปลี่ยนก็พอ)*
      ```json
      {
        "displayName": "John Doe",
        "title": "Senior Frontend Developer",
        "bio": "สวัสดี ยินดีที่รู้จักนะครับ ผมชอบเขียน React มาก!",
        "location": "Bangkok",
        "linkedinUrl": "https://linkedin.com/..."
      }
      ```
*   `GET /users/:username/portfolio`
    *   **ทดสอบอะไร:** ดึงข้อมูล **ทุกอย่าง** ของคนๆ นั้น (ประวัติ, ทักษะ, เหรียญ, โปรเจกต์ Github) เพื่อเอาไปโชว์ในหน้า Resume
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `GET /users/:username`
    *   **ทดสอบอะไร:** ดูข้อมูล Profile ปกติของคนอื่น (ในมุมมองสาธารณะ)
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง

### 3. 🧠 หมวดทักษะ (`/skills`)
จัดการคลังสกิลของเราเอง (เพิ่ม, ลบ, แก้ไข, โชว์คะแนนสอบ)

*   `GET /skills/me`
    *   **ทดสอบอะไร:** ดูว่า "ฉัน" มีการบันทึกทักษะอะไรแปลกๆ ไว้บ้าง
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
*   `GET /skills/:userId`
    *   **ทดสอบอะไร:** แอบไปดูทักษะของคนอื่น (กรอก user ID เข้าไป)
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `POST /skills`
    *   **ทดสอบอะไร:** เพิ่มสกิลใหม่ให้ตัวเอง (ใส่ชื่อทักษะ, หมวดหมู่ และระดับ 1-5)
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "name": "React",
        "category": "Frontend",
        "level": 3
      }
      ```
*   `PUT /skills/:skillId`
    *   **ทดสอบอะไร:** เอาไว้ปรับระดับสกิล หรือแก้ชื่อสกิล
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "name": "React.js",
        "level": 4
      }
      ```
*   `DELETE /skills/:skillId`
    *   **ทดสอบอะไร:** ทดสอบลบสกิลที่เพิ่งใส่พลาดไปทิ้ง
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
*   `POST /skills/:skillId/quiz`
    *   **ทดสอบอะไร:** ทดสอบส่งผลคะแนนทำข้อสอบ (Quiz) ของสกิลนั้นๆ เข้ามาบันทึกเก็บไว้
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):** *(คะแนนที่ได้อย่างเช่น 3 หรือ 4)*
      ```json
      {
        "score": 3
      }
      ```

### 4. 👍 หมวดการรับรองทักษะ (`/endorsements`)
การให้คนอื่นมาการันตี หรือโหวตให้สกิลของเรา

*   `POST /endorsements/request`
    *   **ทดสอบอะไร:** ขอสร้าง Link คำขอโหวต เพื่อเอาไปส่งให้เพื่อนหรือคนอื่นกดให้คะแนนเรา
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "recipientName": "ชื่อเพื่อนที่จะเอาลิ้งไปแปะให้",
        "recipientEmail": "email@เพื่อน.com"
      }
      ```
*   `POST /endorsements/direct`
    *   **ทดสอบอะไร:** ถ้าเรา Login อยู่แล้วเปิดไปเจอโปรไฟล์เพื่อน สามารถกดปุ่มให้การรับรองเพื่อนตรงๆ ผ่านเส้นนี้ได้เลย
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "toUserId": "ไอดีของเพื่อนคนที่เรารับรอง",
        "skills": ["รหัสสกิล1", "รหัสสกิล2"],
        "message": "หมอนี่เขียนโค้ดเก่งมาก ผมรับรองเลยครับ!"
      }
      ```
*   `GET /endorsements/verify/:token`
    *   **ทดสอบอะไร:** เช็คว่า Link คำขอนี้ยังไม่หมดอายุใช่มั้ย
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง (คนภายนอกที่ไม่มีแอปก็เข้ามาเช็คและโหวตได้)
*   `POST /endorsements/submit/:token`
    *   **ทดสอบอะไร:** กด Submit ส่งคะแนนโหวต (ถ้ายอดโหวตรวมกันครบ 3 เสียง ระบบจะออกเหรียญทองให้เลย!)
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "skills": ["รหัสสกิลที่ต้องการโหวต1", "รหัสสกิลที่ต้องการโหวต2"],
        "message": "เยี่ยมยอด"
      }
      ```
*   `GET /endorsements/:userId`
    *   **ทดสอบอะไร:** ดึงประวัติว่าที่ผ่านมามีการรับรองจากใครมาแล้วบ้าง
    *   **ต้อง Login ไหม:** ✅ ต้อง Login

### 5. 🐙 หมวด GitHub (`/github`)
ดึงข้อมูล Repositories และสถิติการเขียนโค้ด

*   `POST /github/connect`
    *   **ทดสอบอะไร:** เอา Access Token ของบัญชี GitHub มาเซฟเชื่อมกับหลังบ้าน
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "token": "ghp_xxxxxxxxxxxxxxxxx"
      }
      ```
*   `POST /github/sync`
    *   **ทดสอบอะไร:** สั่งให้ระบบไปดึง Repos จาก GitHub มาเก็บในหน้าประวัติเรา และช่วยเดาสกิลจากภาษาในโค้ดให้ด้วย
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
*   `GET /github/repos`
    *   **ทดสอบอะไร:** ดึงรายการ Repos ทั้งหมดที่เรายิง Sync สำเร็จไปแล้วมาดู
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
*   `GET /github/dashboard`
    *   **ทดสอบอะไร:** ดึงข้อมูลภาพรวม (Profile + Repos + Skills) เอาไปแสดงบนหน้า Dashboard สวยๆ ทีเดียว
    *   **ต้อง Login ไหม:** ✅ ต้อง Login

### 6. 🏅 หมวดเหรียญตรา (`/badges`)
โชว์เหรียญตรา (Badge) จากความพยายาม

*   `GET /badges/:userId`
    *   **ทดสอบอะไร:** ดูว่าคนๆ นี้มีเหรียญรางวัลรับรองอะไรบ้าง
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง

### 7. 📄 หมวดพิมพ์เอกสาร (`/export`)
สร้าง Resume นามสกุล PDF

*   `POST /export/pdf`
    *   **ทดสอบอะไร:** โยนข้อมูล Portfolio ของเราเข้าไปให้เซิร์ฟเวอร์ตีพิมพ์เป็นไฟล์ PDF เอาไปสมัครงาน
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "profile": { "displayName": "นายทดสอบ โปรแกรมเมอร์" },
        "skills": [ { "name": "React" } ],
        "endorsements": [ { "message": "เยี่ยม" } ]
      }
      ```

### 8. 💼 หมวดหางาน (`/jobs`)
บอร์ดประกาศรับสมัครงานและการจับคู่

*   `GET /jobs`
    *   **ทดสอบอะไร:** ดึงประกาศรับสมัครงานทั้งหมดมาโชว์ในหน้ารวม
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `GET /jobs/recommendations`
    *   **ทดสอบอะไร:** ให้ระบบพิจารณาหางานที่ "ตรงกับสกิล" ที่ตัวคุณมี มานำเสนอ
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
*   `GET /jobs/:id`
    *   **ทดสอบอะไร:** ดูข้อมูลรายละเอียดเนื้องานและคุณสมบัติของงานนั้นๆ (กดดูเพิ่มเติม)
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `POST /jobs/:id/apply`
    *   **ทดสอบอะไร:** ยิงเพื่อกดปุ่ม "สมัครงาน" (สามารถส่ง Cover Letter แนบไปได้ด้วย)
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "coverLetter": "สวัสดีครับคุณ HR ผมมีความสนใจจะร่วมงานกับบริษัทคุณเพื่อต่อยอดทักษะ..."
      }
      ```

### 9. ⚔️ หมวดความท้าทาย (`/challenges`)
รับโจทย์ตอบคำถาม เพื่อทดสอบความรู้และได้คะแนนเพิ่ม

*   `GET /challenges`
    *   **ทดสอบอะไร:** ดึงโจทย์การท้าทายทั้งหมดที่มีในระบบ
    *   **ต้อง Login ไหม:** ❌ ชิมลางได้ ไม่ต้อง
*   `POST /challenges`
    *   **ทดสอบอะไร:** สร้างโจทย์ Challenge คำถามข้อใหม่ขึ้นมา
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "title": "โจทย์แก้บักใน React",
        "description": "จงแก้รหัสในฟังก์ชั่นให้ถูกต้อง...",
        "relatedSkill": "React"
      }
      ```
*   `POST /challenges/submit`
    *   **ทดสอบอะไร:** ยิงส่งคำตอบของโจทย์เข้าไปเพื่อรอตรวจ
    *   **ต้อง Login ไหม:** ✅ ต้อง Login
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "challengeId": "ไอดีของโจทย์ (นำมาจาก GET)",
        "submissionUrl": "https://github.com/my/repo/pull/1"
      }
      ```
*   `GET /challenges/verify-info/:token`
    *   **ทดสอบอะไร:** เรียกดูข้อมูลของ Challenge จาก Link/Token ตอนที่จะเริ่มทำโจทย์
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
*   `POST /challenges/verify/:token`
    *   **ทดสอบอะไร:** ทำหน้าที่ตรวจคำตอบของโจทย์ข้อนั้นๆ
    *   **ต้อง Login ไหม:** ❌ ไม่ต้อง
    *   **ตัวอย่าง Body (JSON):**
      ```json
      {
        "comments": "ทำได้ดีมากครับ โค้ดคลีนอ่านง่าย"
      }
      ```

## 🎯 สรุปวิธีเริ่มเทส (ถ้ายังงง ลองทำตามนี้ดูครับ)

1. เปิดเครื่องเซิร์ฟเวอร์ Backend แล้วไปที่: `http://localhost:3000/swagger`
2. ทดสอบ API ที่ **ไม่ต้อง Login** เช่น ลองคลิกไปที่หมวด `/jobs` เลือกลูกศรลง `GET /jobs` กด `Try it out` และกด `Execute` ตรงๆ ได้เลย ดูผลลัพธ์ว่าแสดงรายการงานออกมามั้ย
3. ขยับมาทดสอบ API แบบ **ต้อง Login** 
   - ให้เปิด Frontend ไปลองล๊อกอินให้เรียบร้อยก่อน
   - ค้นหา Token ของตัวเอง (อาจดูจาก Network แท็บตอนยิง API ครั้งแรก หรือ `console.log`)
   - นำ Token ไปแปะในแม่กุญแจ 🔓 ตัวใหญ่ๆ ของหน้า Swagger
   - หลังจากนั้นจำลองหน้าตาของข้อมูล (JSON Body) จากโค้ดด้านบน ไปใส่ในช่องก้อนคำร้องได้เลยครับ!
